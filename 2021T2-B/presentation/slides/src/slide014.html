Memory Mapping Emulation Challenge

The first challenge is the emulation is all in the user space, we can't modify the paging structures.

The second challenge we want the program to continue execution normally after a seL4 mapping invocation.

The third challenge is emulating the lazy mapping, so we don't map a page until we touch the page.

The current implementation uses signal to solve the challenges:

  • signals are software interrupts, they are well mapped to the seL4 VM fault emulation.
  • signal handlers can do all the mapping jobs implicitly.
  • after returning from signal handlers the program execution. continues normally.
  • emulating both explicit mapping call or lazy page mapping will work.

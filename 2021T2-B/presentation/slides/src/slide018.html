seL4 Kernel Recap

Recap on seL4 kernel functionalities

For emulating a simple hello world roottask, the minimal requirements of the kernel objects are cnode, vspace, endpoints, and tcb.

  • capability: access rights + reference. With the capability, we can retrieve the virtual address of any kernel object and access the object.
  • vspace: architecture-dependent paging structures.
  • the endpoint facilitates the message-passing communication between threads.
  • the tcb represents the seL4 thread structure. (thread state, capability pointer to the IPB buffer, fault, scheduling priority, etc.)

Main takeaway:

Kernel accessing all the kernel objects using virtual addresses as a reference, so we need to emulate the physical memory mapping area in the kernel window.

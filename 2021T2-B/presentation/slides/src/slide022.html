Future Work

The current implementation solved most of the fundamental problems. And can emulate simple roottask and most of the kernel functionalities. (cnodes, vspace, tcb, etc.)

However, due to the time limits (I've been blocked on some problems and bugs for quite a long time) and my knowledge. There are still lots of work that haven't been done yet.

Still working on it, might be finished soon

  • About emulating scheduling, I've just come up with the idea recently, still implementing it.
    • To emulate thread suspension, we can use blocking sockets, so each time kernel uses the scheduling algorithm to determine the next running seL4 thread and only replies to it. Hence all other seL4 threads will block on the socket except one that has been chosen.
    • To emulate the preemption, we can set a timer in the kernel emulator, and signal our seL4 application, then the signal handler routine is invoked and block on the socket until the next IPC message is passed from the kernel emulator telling it to resume.
  • User-configurable kernel booting emulation. (rootask image, physical memory size, platform information, etc.)
  • Running multiple seL4 threads. (Partially done)
  • Once the previous step is done, we should be able to run CAmkES on the emulation framework.

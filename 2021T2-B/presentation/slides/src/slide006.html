Minimusllibc Implementation

Minimusllibc is a C library only used by the emulation library.

Implementation challenges

  • can't directly link the glibc on the host due to symbol name confliction. ✗
  • can't load the library at runtime as dynamic loading library depends on glibc. ✗

Current solution, simple but works :)

  • port musllibc (glibc is too complex, porting it is overkill) and change symbol names. (named minimusllibc) ✓

The current minimusllibc provides

  • socket related APIs such as mini_socket, mini_accept, etc..
  • I/O APIs such as mini_write, mini_read, etc.
  • memory mapping APIs such as mini_mmap, etc.
  • signal APIs such as mini_sigaction, mini_sigstack, etc.
The minimusllibc is not thread safe for the simplicity.
